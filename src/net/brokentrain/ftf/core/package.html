The core package is responsible for providing the model of the system. Various behaviours of the system are delegated to one or more subpackages depending on the service required. Additionally, various data structures that were required for different operations of these services reside here.

At the heart of Fetcher is the ability to start one or more {@link net.brokentrain.ftf.core.Source Sources}, which are responsible for traversing a given URL and obtaining results from this page.

A result can be identified in different ways. Any result must implement the {@link net.brokentrain.ftf.core.DataStore DataStore} interface in order to see what it can optionally provide to a user. The three different interfaces are mapped below:
InterfaceConcrete Implementation
{@link net.brokentrain.ftf.core.URLStore URLStore} {@link net.brokentrain.ftf.core.URLData URLData}
{@link net.brokentrain.ftf.core.FileStore FileStore} {@link net.brokentrain.ftf.core.FileData FileData}

Additionally, the {@link net.brokentrain.ftf.core.SourceManager SourceManager} is responsible for managing these sources.

The {@link net.brokentrain.ftf.core.Config Config} and related {@link net.brokentrain.ftf.core.ConfigLoader ConfigLoader} objects handle user configuration.
