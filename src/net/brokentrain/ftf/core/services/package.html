The services sub-package is responsible for providing implementation of the different providers that the system supports. These providers may use their own concrete API or be simple web page or XML parsers depending on their natue.

The following services are supported by default:
ServiceDescription
ArXivProvider Basic wrapper around the ArXiv lookup service.
CiteSeer Web scrapper to the CiteSeer online service that incorporates a wide range of academic sources.
DOIProvider Basic wrapper around the DOI lookup service.
GoogleScholar Web scrapper to the scholar service provided by Google.
GoogleWebSearch API service for Googles web search service. Please note that a suitable key for the API must be set in the user configuration files.
PlosJournals Web scrapper to the biology service provided by the Public Library of Science.
PubMed Interface to the EUtilities tools provided by PubMed.
PubMed Central Web scrapper to the PubMed Central service provided by PubMed.
Scirus Web scrapper to the Scirus service.
Terrier API service for the Terrier desktop application. Please note that further configuration is possible within the user configuration files.
WebOfKnowledge Web scrapper to the Web of Knowledge service provided by Thomson.
YahooWebSearch API service for Yahoos web search service. Please note that a suitable application ID for the API must be set in the user configuration files.

The {@link net.brokentrain.ftf.core.services.ServiceFactory ServiceFactory} class handles the registration and management of individual services. Any provider wishing to register itself must provide the following code: 


    public class MyProvider implements SearchService {

    private static class Factory extends ServiceFactory {
        protected SearchService create() {
            return new MyProvider();
        }
    }

    static {
        ServiceFactory.addFactory("MyProvider", new Factory());
    }
and can then be used normally.
